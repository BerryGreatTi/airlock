# App Bundle

A macOS application package -- a directory with a `.app` extension that Finder presents as a single clickable icon. Internally it follows a fixed structure:

```
Airlock.app/Contents/
  Info.plist              Metadata: bundle ID, version, executable name, icon file
  MacOS/
    AirlockApp            Main executable (Swift GUI)
    airlock               Helper executable (Go CLI engine)
  Resources/
    AppIcon.icns          Application icon at multiple resolutions
  _CodeSignature/         Code signing artifacts (generated by codesign)
```

## Airlock's two-binary bundle

Airlock bundles two executables in `Contents/MacOS/`:

| Binary | Language | Role |
|--------|----------|------|
| `AirlockApp` | Swift | GUI front-end. Declared as `CFBundleExecutable` in Info.plist -- macOS launches this when the user opens the app. |
| `airlock` | Go | CLI engine. Performs container orchestration, age encryption, Docker API calls. Invoked as a subprocess by the Swift binary. |

`CLIService.resolveAirlockBinary()` finds the Go binary by checking `Bundle.main.executableURL` sibling path first, then `$PATH`, then `/usr/local/bin/airlock`.

## Code signing

Apps must be signed to run on macOS. Signing levels:

| Level | Tool | Gatekeeper behavior |
|-------|------|---------------------|
| Ad-hoc | `codesign --sign -` | Blocked on download. User must bypass via right-click > Open. |
| Developer ID | `codesign --sign "Developer ID Application: ..."` + `xcrun notarytool` | Accepted after notarization. No warning. |
| App Store | Xcode archive + upload | Distributed via Mac App Store only. Requires sandboxing. |

Airlock uses ad-hoc signing for local builds and targets Developer ID signing for GitHub Releases distribution (see ADR-0009).
